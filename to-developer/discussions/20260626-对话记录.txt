对的部分：

如果你用 OS 把它后台启动，它就是后台进程。

这一轮里，如果 prompt 要求 spawn subagents，Codex runtime 可以在这个 run 里创建 subagent threads。

需要修正的部分：
subagents 不是你手动启动的多个 codex exec 进程。
它们是 Codex runtime 管理的 agent threads，不是稳定承诺为“同一个 OS 进程里的线程”。实现细节别依赖。

leader session = 可恢复的上下文/历史
subagents = 这次 worker run 内部由 Codex 编排的临时并行 agent threads
这正适合你的场景：外部 heartbeat 定时启动 codex exec resume <leader_session>，它醒来跑一轮，必要时内部派 subagents 分析日志/状态，汇总后退出。
