### 2. Executor 角色拆分 ✅

将 Executor 从单一角色拆分为三个专用角色，解决职责混淆问题。

| 角色 | 职责 | Caller | Skill 文件 |
|------|------|--------|-----------|
| **Coder** | 写代码、测试、重构 | executor | `skills/coder/SKILL.md` |
| **Deployer** | SSH 同步、启动训练、监控实验 | executor | `skills/deployer/SKILL.md` |
| **Writer** | 写论文、文档、Rebuttal | executor | `skills/writer/SKILL.md` |

**配套更新：**

- `skills/leader/SKILL.md` — 派发时使用专用角色
- `skills/shared-references/agent-guide.md` — Executor 角色分层说明
- `skills/shared-references/executor-skill-routing.md` — 每个角色的推荐 skills

**效果：**

- Leader 派发更精准：`Agent: model: sonnet, description: "Implement code (Coder)"`
- 减少职责越界：Deployer 不再写代码，Coder 不再 SSH
- 便于权限控制：不同角色可用不同 sandbox

### 3. 上下文记忆本地化 ✅

解决跨会话遗忘问题，实现状态持久化。

**新增文件：**

| 文件 | 作用 |
|------|------|
| `CONTEXT_MEMORY.json` | 机器可读状态，会话启动时自动加载 |
| `skills/shared-references/context-memory-contract.md` | 记忆协议定义 |

**Schema 结构：**

```json
{
  "persistent_context": {
    "active_project": {...},
    "active_experiments": [...],
    "framework_state": {...}
  },
  "working_memory": {
    "recent_decisions": [...],
    "pending_questions": [...],
    "blocked_items": [...]
  },
  "retention_rules": {...}
}
```

**分类规则：**

| 类型 | requires_human | 示例 |
|------|----------------|------|
| 需要人决断 | true | 研究方向变更、止损决策、投稿时机 |
| 可自动继续 | false | 代码实现细节、参数调优、文档格式 |

### 4. 阻塞协议增强 ✅

为 `executor-blocked-protocol.md` 添加绕过策略表和累计阈值。

**绕过策略表（新增）：**

| 阻塞类型 | 绕过方案 1 | 绕过方案 2 |
|---------|-----------|-----------|
| permission | 用 Read 替代 cat | 用 Glob 替代 ls |
| network | 重试 3 次 | 换备用服务器 |
| resource | 减小 batch_size | 用 CPU 跑小规模测试 |
| dependency | 用 conda 安装 | 用系统包管理器 |
| missing_file | 用测试数据 | 从模板生成 |

**累计阈值规则（新增）：**

| 累计阻塞次数 | 行为 |
|-------------|------|
| 1-2 | 继续任务其他部分，跳过阻塞点 |
| ≥ 3 | 整个任务停止，写总报告 |

## 三、三边架构适配

### Claude Code 模式（现有）

```
Leader (Opus, 窗口1)
   └─ Agent(model: sonnet) → Executor (窗口2)
   └─ mcp__codex__codex → Reviewer (GPT-5.5, 窗口3)
```

### Codex CLI 模式（新）

```
# 窗口1
codex exec -p leader "研究任务..."

# 窗口2
codex exec -p executor "实现代码..."

# 窗口3
codex exec -p reviewer "审查代码..."
```

**差异：**

| 特性 | Claude Code | Codex CLI |
|------|-------------|-----------|
| Agent 派发 | `Agent(model, prompt)` | 无等价物 |
| 角色切换 | 自动 | 手动开多窗口 |
| 通信方式 | 内存 + 文件 | 纯文件 |

**当前策略**: 保守路线（多窗口 + profiles），人手动协调
